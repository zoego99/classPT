1 AI 協作四步驟
📌 練習主題：年度工作計畫撰寫（可替換為任何你正在進行的報告或企劃）
【問】釐清思路
使用時機：開始寫任何報告、計畫或企劃前，先請 AI 扮演資深顧問向你提問，把想法整理清楚之後再動筆，省去反覆修改的時間。

把下方 Prompt 中的【】替換成你的任務主題，就可以直接使用。

  • 【任務名稱】→ 例：年度行銷計畫、Q3 業績報告、新人訓練方案
  • 【報告對象】→ 例：主管、董事會、客戶、全公司同仁
  • 【專業角色】→ 例：資深行銷顧問、財務分析師、人資專員
📌 Prompt — 釐清思路
我現在要進行【任務名稱】的撰寫， 報告對象為【報告對象】 在正式產出前，請扮演【專業角色】 協助針對此任務的『目標受眾、核心重點、預期效益』向我提出 5～8 個關鍵問題， 幫助我釐清思路 在我們討論未完成前，先不要開始寫報告
【定】文件生成
使用時機：與 AI 討論清楚後，請它依你的需求正式產出文件內容。
📌 Prompt — 正式文件產出
根據我們剛才討論的內容，產出正式文件。輸出時遵守以下規範： 1. 必須包含『版本資訊』區塊： - 版次：V1.0（初始草稿） - 最後更新日期：標註今日日期 - 資料查證規範： - 引用數據或法規請標註建議查詢來源 - 動態數據（如市場數字、統計資料）標註「需進一步查證」 2. 內容架構： 請依照討論結果，協助擬定完整架構，建議涵蓋： 一、目的與背景　二、現況說明　三、目標設定 四、執行方案與時程　五、所需資源　六、預期效益 3. 輸出格式： 請將完整內容放入程式碼區塊（Code Block）輸出，方便一鍵複製
【定】修改建議
使用時機：文件草稿產出後，讓 AI 扮演審稿者，主動找出需要加強的地方。
📌 Prompt — 修改建議
請針對這份文件草稿，以【資深主管】角色提出修改建議。 請依以下三個面向逐一檢視，每項給出具體修改方向： 1. 邏輯結構：各章節順序是否清晰？論點之間有無跳躍或矛盾？ 2. 文字表達：語氣是否符合場合？有無口語化、語意模糊之處？ 3. 內容完整性：目標、方案、效益是否均衡？有無重要資訊遺漏？ 請以條列方式輸出，每條建議附上對應的段落位置。
【定】反向檢查
使用時機：文件接近定稿前，模擬主管或審查方的挑剔視角，提前找出問題點。
📌 Prompt — 反向檢查
請扮演【嚴格的審查委員】，對這份文件進行反向檢查。 檢查重點： 1. 哪些數據或說法缺乏來源依據，可能在簡報時被質疑？ 2. 執行方案是否說明清楚？責任分工有無模糊地帶？ 3. 若遇到預算不足或時程延誤，文件中是否有應變說明？ 4. 整份文件最薄弱的一個部分是哪裡？建議如何補強？ 請以「審查意見」格式輸出，語氣可以嚴格，目的是幫我在正式呈報前先補好洞。
【骨】邏輯骨架校準
使用時機：文件內容確認後，請 AI 幫你規劃簡報大綱，決定哪頁放什麼、用什麼版面配置。
📌 Prompt — 大綱校準
現在進入【邏輯骨架校準階段】。 請根據我們剛才討論完成的文件內容， 針對【報告對象】擬定簡報大綱。規範： - 規劃約【8～10】張架構 - 每頁須含主題與核心觀點，禁止跳級產出內文或旁白 - 對數據頁面，請標註建議採用「便當盒排版法」 - 對流程頁面，請標註建議採用「蛇形流程」 - 產出後立即停止，等待進一步指令 - 強制使用 Markdown 程式碼區塊輸出
【生】產出簡報

大綱確認後，選擇以下任一方式產出簡報。

建議流程：先用 Google 簡報快速產出架構，確認內容後再考慮 HTML 精緻版。

📌 Google 簡報
邏輯大綱已確認。直接建立 Google 簡報。 - 內容需填充每張投影片之正式文案。 - 產出後，請務必顯示『匯出成簡報』的連結。 - 尺寸：16:9
📌 HTML 程式碼簡報
邏輯大綱已確認。 使用 HTML 與 Tailwind CSS 程式碼製作數位簡報。 - 風格要求：現代簡潔、藍灰色調、專業感。 - 結構規範：數據頁採「便當盒排版」、流程頁採「蛇形流程」， 預留上方 30% 標題空白區。 - 請將結果以程式碼區塊呈現，並同步啟動『預覽 (Preview)』視窗。 - 尺寸：16:9
2 NotebookLM 文件智慧整合應用
將文件上傳到 NotebookLM 後，用以下 Prompt 進行跨文件查詢與分析。適合整理大量文件、會議記錄、報告資料。
文件整理
使用時機：上傳一份或多份工作文件，請 AI 快速提煉重點，不用逐字閱讀。
📌 Prompt — 文件重點整理
請針對上傳的文件，完成以下整理： 1. 用條列方式列出文件的核心重點（每項不超過 3 點） 2. 找出文件中所有出現的數據或關鍵數字，整理成一張表格 3. 列出文件中提到的所有待辦事項或行動方案， 並標注責任單位與期限（若有的話）
📌 Prompt — 快速摘要（給主管看）
請將上傳的文件整理成一份給主管快速瀏覽的摘要。 格式要求： - 總長度不超過半頁 A4 - 開頭一句話說明「這份文件是什麼」 - 三個最重要的重點（條列） - 一個「需要主管決定的事項」（若有） 語氣簡潔，避免廢話。
會議記錄整理
使用時機：上傳多份會議記錄，讓 AI 跨文件找出決議追蹤、重複問題、責任分工。
📌 Prompt — 跨文件會議分析
我上傳了多份不同時期的會議記錄。 請跨文件完成以下分析： 1. 整理所有會議的決議事項清單，標注每項的負責單位與完成期限 2. 找出哪些問題在多次會議中重複出現 3. 哪些決議在後續會議被確認完成？哪些沒有後續追蹤記錄？
跨文件彙整
使用時機：把整個專案的所有文件都上傳，讓 AI 幫你做一次全面盤點。
📌 Prompt — 專案總彙整
請將上傳的所有來源文件視為一個完整專案，進行跨文件總整理： 1. 從開始到現在，時間軸上發生了哪些關鍵事件？ 2. 哪個單位或人員在整個專案中被分配到最多任務？ 3. 若我是剛接手的新主管，要在 3 分鐘內掌握這個專案， 最重要的 5 件事是什麼？請用條列方式說明。
逐字稿整理
整理完成後的用途：建議將整理好的正式會議紀錄存為筆記加入知識庫，後續即可跨文件查詢過去的討論結論。
📌 Prompt — 逐字稿轉會議紀錄
這是一份會議的錄音逐字稿，語氣口語，內容較雜亂。 請協助完成以下整理： 1. 整理成正式會議紀錄格式，包含： 討論事項、各方意見摘要、決議事項、後續追蹤事項 2. 找出所有主管或負責人交辦事項，列出負責人與預計完成時間 3. 整理「問題與改善建議」清單，標注是誰提出的
匯出成品 Prompt
簡報來源建議：在 NotebookLM 中，勾選【大綱骨架 .md 檔】，不勾選其他文件，確保簡報內容聚焦。
📌 簡報草稿（含視覺風格建議）
請根據上傳的大綱文件，製作一份簡報草稿，並依照以下視覺規格標注每頁建議風格： # 角色設定 你是一位資深行政幕僚兼簡報設計顧問。 # 簡報規格 - 總頁數：10 張，不可多不可少 - 尺寸：16:9 # 每頁輸出格式 ### 投影片第 X 張：[標題] - 視覺風格建議：[從以下擇一] - 數據頁 → 現代數據儀表板風（淺色儀表板＋數字卡片 KPI） - 流程頁 → 等距視角微縮風格（流程路徑圖） - 多重並陳頁 → 便當盒排版（大小格混排） - 封面/結語頁 → 極簡向量插畫風格（三色限定） - 版面配置：[便當盒 / 蛇形流程 / 中心對焦，擇一] - 核心列點：[3 點，每點不超過 20 字] - 講師講稿：[80 字內，語氣簡潔專業] # 注意事項 所有內容必須 100% 基於上傳的大綱來源，禁止憑空新增資訊。
📌 心智圖結構
請根據上傳的所有來源文件，產出一份心智圖結構（純文字版）。 格式規範： - 中心主題：[文件的核心主題名稱] - 第一層分支（3～5 個）：主要工作主軸或專案階段 - 第二層分支：每個主軸下的關鍵任務或重點 - 第三層分支：具體數據、負責單位或執行細節 請以縮排條列方式輸出，方便複製至 XMind 或 Miro 使用。
📌 語音摘要腳本（雙主持人對話版）
請根據上傳的大綱，製作一份「上台報告語音摘要」。 # 角色設定 主持人 A：資深業務主管（語氣沉穩、有條理） 主持人 B：熟悉執行細節的承辦人（語氣親切、善用具體例子） # 風格要求 - 對話式進行，兩人交替說明各章節重點 - 主持人 B 適時舉出具體情境或案例 - 語氣專業但不生硬，總長度約 5 分鐘可聽完 # 內容規範 - 開場：點出這份報告的核心價值 - 中段：依主要章節各用約 1 分鐘說明亮點 - 結尾：點出最值得帶走的 1 個重點或行動建議

如何使用這個 Prompt

以下 Prompt 框架可套用到任何主題的資訊圖製作。使用前請替換【】中的內容：

  • 【主題描述】→ 你的資訊圖主題，例如：職場時間管理五大原則
  • 【應用場景】→ 例如：教育科普、社群行銷、企業簡報
  • 【重點內容 1~3】→ 你要放進圖裡的三個核心訊息

通用資訊圖 Prompt 框架
📌 Prompt — 資訊圖（橫向 16:9）
請生成一張【橫向 16:9】資訊圖，已包含完整排版與視覺設計。 主視覺 / 構圖內容描述： 針對「【應用場景】」與「【主題描述】」主題。 畫面採便當盒排版，整齊並列三個圓角格子， 每個格子內含對應主題的平面向量插圖與說明文字。 視覺風格：極簡向量插畫風格 - 扁平化設計，無陰影無漸層 - 色彩明快，每個格子用不同強調色區分 版面配置：便當盒排版（模組化分割） 簡報母片設定：預留上方 30% 標題空白區 文字設計細節： - 主標題：【主標題文字】 - 副標題：【副標題文字】 - 重點資訊： 1. 【重點內容 1】 2. 【重點內容 2】 3. 【重點內容 3】 - CTA：【行動呼籲或補充說明】 配色方案：白色底（#FFFFFF）＋深藍主色（#2A3F6F）＋亮藍強調（#4A90E2）＋淺米輔助（#F8F6F1）

4 數據匯整
把紙本、照片、截圖等資料交給 AI 整理成表格，再貼入 Excel 使用。
照片／紙本資料轉 Excel
操作習慣：整理完成後，用【Ctrl + C】複製 AI 回覆的表格，再到 Excel 按【Ctrl + V】貼上，不需要手動輸入。
情境原始格式整理後欄位（範例）
活動報名表紙本表單照片姓名／部門／報名項目／備註
出勤紀錄手寫打卡表日期／姓名／到班時間／離班時間／備註
客戶意見回饋便利貼或紙本問卷日期／意見分類／內容摘要／優先程度
庫存盤點表手寫清冊品項名稱／規格／數量／位置／備註
❌ 常見問法（不佳）：「請把這張圖片轉成 Excel」
AI 不知道你要什麼欄位、不知道格式、不知道用途，只能猜。
📌 好 Prompt — 紙本資料轉表格
請協助整理圖片中的【表單名稱】內容。 請根據圖片資訊，整理成「可直接貼到 Excel 的表格格式」。 欄位請包含： - 【欄位 1】 - 【欄位 2】 - 【欄位 3】 - 【欄位 4】 - 備註 請用「表格」方式輸出，不要加入多餘說明文字。 若圖片中某欄位無資料，請填寫「無」。

使用提醒：把上方 Prompt 的【】換成你的實際欄位名稱，例如姓名、部門、日期、數量等，AI 就會照你的規格整理。

問卷分析
使用建議：將問卷原始資料（CSV 或表格）上傳給 AI，或貼入 NotebookLM，再用以下 Prompt 進行分析。
📌 Prompt — 基礎統計
請幫我整理這份問卷的基本結果： 1. 各選項的填答人數與百分比分佈 2. 哪個選項獲得最多人選擇？ 3. 開放題的回答中，出現最多次的關鍵詞是哪些？ 4. 整體滿意度（若有）的平均分數是多少？
📌 Prompt — 交叉分析
請針對這份問卷進行以下交叉分析，並以表格呈現結果： 1. 【分組條件 A】× 【評估指標 B】（平均分或百分比） 2. 【分組條件 C】× 【評估指標 D】 3. 找出哪個群體的滿意度最高？哪個最低？ 分析完成後，請用 2～3 句話說明最值得關注的發現。
📌 Prompt — 主管報告摘要
請扮演【資深分析顧問】，根據這份問卷資料， 幫我撰寫一份給主管看的「問卷調查摘要報告」。 格式要求： - 總長度不超過一頁 A4 - 包含：重要數據亮點、三個主要發現、三個改善建議 - 語氣簡潔專業 - 數據需從問卷資料中取得，不可憑空編造
