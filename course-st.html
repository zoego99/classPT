1 AI 協作四步驟
📌 練習主題：新機型診所推廣與售後服務方案
【問】釐清思路
使用時機：開始寫任何報告前，先請 AI 扮演資深幕僚向你提問，幫助你把思路整理清楚，再開始動筆。
📌 Prompt — 釐清思路
協助規劃「新機型診所推廣與售後服務方案」， 但我還不清楚應該聚焦哪些面向。 請先以【資深業務顧問】角色，對我提出 5 個問題，幫助我確認： 1. 這份規劃的主要受眾（總經理？業務團隊？診所端？） 2. 最需要解決的服務瓶頸是什麼 3. 這次規劃要有哪些必要產出（報告、SOP、簡報？） 請一次提出所有問題，等我回答後再開始。
【定】報告生成
📌 Prompt — 正式報告產出
根據我們剛才討論的內容，產出正式文件。輸出時遵守以下規範： # 角色設定 你是一位資深醫美儀器產業顧問，擅長客戶服務流程設計與新品推廣管理。 # 任務 針對「新機型診所推廣與售後服務方案」進行規劃。 # 輸出格式 請以 Markdown code block 輸出，結構如下： ## 一、計畫背景與挑戰 ## 二、三大服務優化方向（依據我們確認的三個方向） ## 三、各部門任務分工表（Markdown 表格） ## 四、關鍵 KPI 設定（僅列指標名稱與目標方向，不捏造數字） # 邊界規範 - 所有內容必須來自剛才的對話，禁止自行補充未討論的內容 - 每個章節以條列呈現，每點不超過 30 字 - 禁止捏造具體數據（KPI 僅寫「待填入」或「依實際設定」） - 部門分工表欄位：部門、任務、負責人、期限
【定】定標準三要素
定標準的三個關鍵要素：角色設定（決定語氣與專業度）、格式鎖定（指定輸出結構）、邊界聲明（明確禁止捏造數據）。
📌 Prompt — 反向查證
在你開始填充內容之前，請先做反向查證： 1. 逐一確認每個章節，是否有資料來源支撐（來自上傳文件） 2. 列出「無法從來源文件找到依據」的項目，標注 ⚠️ 3. 以上確認完成後再問我：「是否繼續生成？」
【生】反向查證角色變化版
使用時機：報告接近定稿前，模擬不同角色的挑剔視角，提前補洞。
📌 Prompt — 角色變化反向查證
例1：請扮演嚴格的主管，針對這份新品推廣企劃提出 3 個可能需要改善的問題 例2：請以診所端角度，檢查這份儀器操作教學簡報是否有不容易理解的地方 例3：請以審查委員角度，指出這份成果報告可能缺少哪些數據或說明
【骨】邏輯骨架校準
📌 Prompt1 — 大綱校準
現在進入【邏輯骨架校準階段】。 請根據我們剛才完成的「新機型診所推廣與售後服務方案」報告， 針對【公司總經理與各部門主管】擬定簡報大綱。 規範： - 規劃約【10】張架構 - 每頁須含主題與核心觀點，禁止跳級產出內文或旁白 - 數據頁面請標註建議採用「便當盒排版法」 - 產出後立即停止，等待進一步指令 - 強制使用 Markdown code block 區塊輸出
📌 Prompt2 — 大綱校準（對應視覺規格）
# 任務進度：【簡報大綱校準階段】 ## 銜接背景與角色設定 - **資料來源**：依據我們剛才完成的「新機型診所推廣與售後服務方案」報告 - **AI 角色**：資深業務幕僚，協助將報告轉化為具說服力的簡報邏輯 - **簡報對象**：公司總經理與各部門主管，偏好數據導向、重視執行可行性 ## 簡報規格參數 - **上台時間**：15 分鐘 - **投影片張數**：約 10 張 - **輸出格式**：以「投影片 [編號]：[標題]」呈現 - **每頁必含**：核心論點摘要 + 建議視覺化圖型 ## 參、視覺與版面規格 - **整體風格**：現代數據儀表板風（淺色介面、專業 KPI 數字卡片） - **色系**：三色限定 主色：深海藍 #1E3A5F 輔色：香檳金 #C9A84C 底色：暖米白 #F5F0E8 - **視覺質感**：無陰影、無漸層、幾何平整 - **封面設計**：強制預留上方 30% 標題空白區 - **內頁排版**：複雜數據頁標註「便當盒排版法」或「網格矩陣」 ## 強制約束 - 禁止產出具體簡報文案、旁白或生圖指令 - 僅輸出邏輯骨架，產出後立即停止，等待指令 ## 格式鎖定 - 強制以 Markdown 程式碼區塊輸出大綱 「報告背景已備齊，請開始擬定簡報邏輯大綱。」
【生】產出簡報
📌 Google 簡報
邏輯大綱已確認。直接建立 Google 簡報。 - 內容需填充每張投影片之正式文案 - 聚焦於業務部與各協作部門的任務分工與 KPI 設定 - 呈現對象：公司總經理與各部門主管 - 產出後請顯示「匯出成簡報」的連結 - 尺寸：16:9
📌 HTML 程式碼簡報
邏輯大綱已確認。 使用 HTML 與 Tailwind CSS 程式碼製作數位簡報。 - 視覺風格：現代企業品牌風（深金 + 深藍 + 米白） - 結構規範：任務分工採用【便當盒排版】，KPI 採用【數字卡片版型】 - 請將結果以程式碼區塊呈現，並同步啟動「預覽 (Preview)」視窗 - 尺寸：16:9
2 NotebookLM 文件智慧整合應用
上傳文件到 NotebookLM 後，用以下 Prompt 進行跨文件查詢與分析。
報告整理
📌 Prompt — 成果報告整理
請針對上傳的「新品上市客戶服務整合計畫」成果報告， 以【資深業務幕僚】角色完成以下整理： 1. 用條列方式列出三大工作主軸的核心成果（每項不超過 3 點） 2. 找出報告中所有出現的數據，整理成一張表格 3. 列出「問題與改善建議」章節提到的所有待辦事項， 並標注責任部門與期限
會議記錄整理
📌 Prompt — 跨文件會議分析
我上傳了四份不同時期的會議記錄（第1次籌備、第2次籌備、上市期間滾動調整、結案檢討）。 請跨文件完成以下分析： 1. 整理四次會議的決議事項清單，標注每項的負責部門與完成期限 2. 找出哪些問題在多次會議中重複出現 3. 哪些初次會議的決議，在後續會議被追蹤到完成？哪些沒有提到後續？
總專案彙整
📌 Prompt — 跨文件總整理
請將上傳的所有來源文件（成果報告＋四份會議記錄）視為一個完整專案， 進行跨文件總整理： 1. 這個專案從籌備到結案，時間軸上發生了哪些關鍵事件？ 2. 哪個部門在整個專案中被分配到最多任務？ 3. 若我是新任主管，要在 3 分鐘內掌握這個專案， 最重要的 5 件事是什麼？請用條列方式說明。
逐字稿整理
整理完成後的用途：建議將正式會議紀錄存為筆記加入知識庫，後續即可跨文件查詢——「這個問題上次討論結論是什麼？」
📌 Prompt — 逐字稿轉會議紀錄
這是一份新品上市結案檢討會議的錄音逐字稿，語氣口語，內容較雜亂。 請協助完成以下整理： 1. 整理成正式會議紀錄格式，包含： 討論事項、各部門意見摘要、決議事項、後續追蹤事項 2. 找出所有主管交辦事項，列出負責人與預計完成時間 3. 整理「問題與改善建議」清單，標注是哪個部門提出的
匯出成品 Prompt
簡報來源建議：勾選【大綱骨架 .md 檔】，不勾選其他文件
📌 簡報：結合 Nano Banana 視覺風格
請根據上傳的「新品上市客戶服務整合計畫大綱」， 製作一份簡報草稿，並依照以下視覺規格標注每頁建議風格： # 角色設定 你是一位資深業務幕僚兼簡報設計顧問。 # 簡報規格 - 總頁數：10 張，不可多不可少 - 尺寸：16:9 # 每頁輸出格式 ### 投影片第 X 張：[標題] - 視覺風格建議：[從以下擇一] - 數據頁 → 現代數據儀表板風（淺色儀表板＋數字卡片 KPI） - 流程頁 → 等距視角微縮風格（流程路徑圖） - 多重並陳頁 → 便當盒排版（大小格混排） - 封面/結語頁 → 極簡向量插畫風格（企業曼非斯風、三色限定） - 版面配置：[便當盒 / 蛇形流程 / 中心對焦，擇一] - 核心列點：[3 點，每點不超過 20 字] - 講師講稿：[100 字內，語氣沉穩專業] # 注意事項 所有內容必須 100% 基於上傳的大綱來源，禁止憑空新增資訊。
📌 心智圖結構
請根據上傳的所有來源文件，產出一份心智圖結構（純文字版）。 格式規範： - 中心主題：新品上市客戶服務整合計畫 - 第一層分支（3～5 個）：主要工作主軸或專案階段 - 第二層分支：每個主軸下的關鍵任務或重點 - 第三層分支：具體數據、負責部門或執行細節 請以縮排條列方式輸出，方便複製至 XMind 或 Miro 使用。
📌 資訊圖
請根據上傳的成果報告，整理出適合製作資訊圖的核心內容， 並依照以下格式輸出，讓我可以直接貼入 AI 生圖工具： # 資訊圖內容整理 - **主標題**（10 字內）： - **副標題**（20 字內）： - **核心數據**（3～5 個關鍵數字，每個附簡短說明）： - **三大重點訊息**（每點 15 字內）： # 視覺風格建議 - 應用場景：企業簡報與商業提案 - 建議風格：現代數據儀表板風（淺色儀表板＋數字卡片 KPI） - 版面配置：便當盒排版（大小格混排） - 配色方案：三色限定，建議以深金、白、深藍為主色 - 簡報母片設定：預留上方 30% 標題空白區 - 尺寸：16:9
📌 語音摘要腳本
請根據上傳的簡報大綱，製作一份「上台報告語音摘要」。 # 角色設定 主持人 A：資深業務主管（語氣沉穩、重視數據與成效） 主持人 B：熟悉現場的客服領班（語氣親切、善用具體案例） # 風格要求 - 對話式進行，兩人交替說明各章節重點 - 主持人 B 在說明時，適時舉出現場具體情境 - 幽默但不失分寸，總長度約 5 分鐘 # 內容規範 - 開場：點出這份計畫的核心價值 - 中段：依三大主軸各用 1 分鐘說明亮點成果 - 結尾：點出最值得帶走的 1 個改善建議

生成此圖使用的 Prompt
📌 主題：儀器日常保養三大重點，延長使用壽命！
請生成一張【橫向 16:9】資訊圖，已包含完整排版與視覺設計。 **主視覺 / 構圖內容描述**： 畫面右側整齊並列三個圓角的便當盒格子（Bento Box），格子內分別帶有平面幾何向量感（SVG Icon）的「探頭清潔」「定期校正」「妥善收納」插圖。畫面左側由角色「技術顧問」手持平板進行說明，視線自然專注地望向右側的便當盒格子，形成流暢的視覺引導。 **視覺風格**： 採用「極簡向量插畫風格」風格（精準資訊視覺化）。幾何平整、線條簡潔，適合企業報告與數據說明。 - 可愛粉嫩風（線條圓潤，色調粉嫩溫馨，加入卡通圖案元素） - 莫蘭迪色調（低飽和、灰調柔和，優雅沉穩） **版面配置**： 採用「便當盒排版（模組化分割）」與「左右對比」排版方式。 **文字設計細節**： - 主標題：儀器日常保養三大重點，延長使用壽命！ - 副標題：正確保養，效能不打折 - 重點資訊： 1. 探頭清潔：每次使用後立即擦拭，避免殘留物影響效能。 2. 定期校正：依原廠建議週期送回校正，確保輸出穩定。 3. 妥善收納：避免潮濕與高溫環境，使用原廠收納盒存放。 - CTA：定期保養，效果更長久！ **輸出規格要求**： - 比例為橫向 16:9（投影片、簡報背景圖） - 整體構圖飽滿，角色姿態與右側圖文版面具有強烈互動感，配色和諧，字體清晰易讀。

請生成一張【海報 4:3】宣導海報，已包含完整排版與視覺設計。 **視覺風格**： 採用「3D 黏土質感」風格（3D 立體造型）。手工捏製感的 3D 造型，溫暖可愛，現代插畫趨勢。 - 平面插畫上色（柔和漸層，乾淨線條） - 專業形象為主（形象專業且溫暖，具備親切且具信賴感的調性） - 圓潤有機造型（所有卡片與文字背景底襯皆採流暢大圓角設計，避免尖銳稜角） **簡報母片設定**： - AI 自動優化排版（不刻意留白，文字內容與角色比例均勻分佈）。 **文字設計細節**： - 主標題：新品上市，肌膚煥新從這裡開始 - 副標題：專業科技，值得信賴 - 重點資訊： - 核心提醒：全新技術升級，舒適度與效果同步提升。 - 特色摘要：更短療程時間／更低不適感／更穩定效果 - CTA：立即預約體驗，搶先感受新品魅力！ **輸出規格要求**： - 比例為海報 4:3（橫式海報、宣傳單） - 角色人物與左側圖文排版完全分離不重疊。
