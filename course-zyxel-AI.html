1 個人化與連結應用
📌 Gemini／ChatGPT 都推出了「連結應用程式」功能，讓 AI 參考 Gmail、日曆、相簿等個人資料，提供更貼近情境的回覆。開始學 Prompt 之前，先了解這個功能怎麼用、風險在哪。
連結應用程式比較：Gemini vs ChatGPT
項目Gemini（個人化智慧服務）ChatGPT（Connectors）
設定位置點選個人頭像 →「個人化智慧服務」設定 →「Connectors」
可連結應用Gmail、YouTube、Google 相簿、日曆、Workspace 文件Gmail、Google Calendar、Google Contacts、Google Drive 等
預設狀態關閉，需自行開啟並指定要連結的應用關閉，需自行開啟並指定要連結的應用
帳號限制僅限「個人 Google 帳戶」，企業 Workspace 帳號無法使用需要付費方案（Plus／Pro／Team／Enterprise）
訓練用途僅用「提示與生成回覆」做模型訓練，不直接用信件內容執行寄信／改行程等動作前，都需要使用者人工確認
兩家邏輯一致：讀取資料前都要你手動開啟連結；要「做動作」（寄信、加行程）之前，都會停下來讓你確認，不會自己動手。
情境練習：出差拜訪客戶前的「個人化雷達」
使用時機：業務出差拜訪客戶前，開啟 Gemini 的個人化連結，讓 AI 像雷達一樣自動掃描 Gmail、日曆、相簿，抓出你真正需要準備的資料，不用自己一個分頁一個分頁找。
📌 Prompt — 出差行程與信件彙整（連結 Gmail＋日曆）
請根據我的 Gmail 與日曆，整理出這週三「新竹科學園區客戶拜訪」的相關資訊： 1. 找出跟這次拜訪相關的所有信件，整理成重點摘要（含對方提出的需求或疑慮） 2. 確認當天前後的行程安排，提醒是否有時間衝突 3. 列出我這次拜訪前，還有哪些事項尚未回覆或處理
📌 Prompt — 設備安裝照片彙整（連結 Google 相簿）
請從我這個月拍攝的設備安裝／保養照片中， 整理出可以用在客戶教育訓練簡報的素材： 1. 依拍攝地點與設備型號分類 2. 挑出畫面清晰、適合展示安裝流程的照片 3. 每張照片建議搭配一句簡短的簡報說明文字
⚠️ 提醒：這類連結會用到公司客戶資料與信件內容，建議先確認公司 IT／資安部門對企業帳號使用個人化連結的政策，這正是「資料隱私原則」的實際應用。
2 AI 協作四步驟
📌 練習主題：【隼鷹計畫】——新一代資安防火牆上市的客戶溝通與服務整合計畫
💡 隼鷹計畫：像老鷹一樣居高臨下、盯緊每一筆異常流量的新款資安防火牆，代號取「眼觀四方、動作迅猛」之意
詢問原則（GPS 三步驟）
字母代表核心提問
GGoal 目標我要達成什麼成效？
PPosition 背景資料我現在的情境、資源、限制是什麼？
SShape 成果格式最後要交給我的長什麼樣子？
GPS 三步驟：【問】這一步可以用 GPS 三步驟來問清楚——G目標、P背景、S格式，就像開導航，先想終點、再盤點起點、最後定交通方式。這是【問定骨生】四步驟中「問」的具體操作手冊。
【問】釐清思路（GPS：G目標＋P背景）
使用時機：開始寫任何報告前，先請 AI 扮演資深顧問向你提問，幫助你把思路整理清楚，再開始動筆。
⚠️ AI 幻覺：AI 產生看似合理，但實際上錯誤、不存在或未經查證的內容。使用 AI 應注意：AI 結果需人工確認／數據需核對來源。
📌 Prompt — 例1（聚焦 G：目標）
我目前的任務是：「撰寫一份【隼鷹計畫】（新一代資安防火牆）新品上市客戶溝通計畫的成果報告」。 在你開始動手之前，請先以【資深科技產業顧問】的角色， 對我提出 5 個問題，協助我釐清這份報告的目標、受眾與重點。
📌 Prompt — 例2（同時涵蓋 G＋P：目標與背景）
協助規劃「【隼鷹計畫】新機型資安防火牆推廣與售後服務方案」， 但我還不清楚應該聚焦哪些面向。 請先以【資深業務顧問】角色，對我提出 5 個問題，幫助我確認： 1. 這份規劃的主要受眾（公司高層？投資人？經銷商客戶？） 2. 最需要解決的服務瓶頸是什麼 3. 這次規劃要有哪些必要產出（報告、SOP、簡報？） 請一次提出所有問題，等我回答後再開始。
【定】報告生成
Markdown（MD）結構化：
  • 運用 MD 與 AI 工具對話，層級分明可清楚表達格式需求，讓 AI 生成內容更具結構化、省時且易讀。
  • 語法：【# 標題層級 (#、##、###)】、【- 項目符號】、【**粗體** 強調重點】
📌 Prompt — 正式報告產出
根據我們剛才討論的內容，產出正式文件。輸出時遵守以下規範： # 角色設定 你是一位資深科技產業顧問，擅長客戶服務流程設計與新品推廣管理。 # 任務 針對「【隼鷹計畫】新機型資安防火牆推廣與售後服務方案」進行規劃。 # 輸出格式，結構如下： ## 一、計畫背景與挑戰 ## 二、三大服務優化方向（依據我們確認的三個方向） ## 三、各部門任務分工表（Markdown 表格） ## 四、關鍵 KPI 設定（僅列指標名稱與目標方向，不捏造數字） # 邊界規範 - 所有內容必須來自剛才的對話，禁止自行補充未討論的內容 - 每個章節以條列呈現，每點不超過 30 字 - 禁止捏造具體數據（KPI 僅寫「待填入」或「依實際設定」） - 部門分工表欄位：部門、任務、負責人、期限
【定】定標準三要素
定標準的三個關鍵要素：角色設定（決定語氣與專業度）、格式鎖定（指定輸出結構）、邊界聲明（明確禁止捏造數據）。
📌 Prompt — 反向查證
在你開始填充內容之前，請先做反向查證： 1. 逐一確認每個章節，是否有資料來源支撐（來自上傳文件） 2. 列出「無法從來源文件找到依據」的項目，標注 ⚠️ 3. 以上確認完成後再問我：「是否繼續生成？」
【生】反向查證角色變化版
使用時機：報告接近定稿前，模擬不同角色的挑剔視角，提前補洞（紅隊測試 Red Teaming）。
📌 Prompt — 角色變化反向查證
例1：請扮演嚴格的內部審查主管，針對這份新品推廣企劃提出 3 個可能需要改善的問題 例2：請以廠商客戶端角度，檢查這份操作教學簡報是否有不容易理解的地方 例3：請以投資人角度，指出這份成果報告可能缺少哪些數據或說明
【骨】邏輯骨架校準（GPS：S格式）
▶️ 使用時機：

內容確認後，進入骨架規劃。這一步只產大綱，不急著做出成品。

  • AI 很容易「跳級」，稱之為「AI 會過度積極」——你只要一個簡報大綱，它卻給你整份簡報。
  • 【骨】的關鍵是明確指令：只出邏輯大綱，不寫內文。
▶️ 簡報時間與張數參考
  • 通常 1 分鐘對應 1~1.5 張投影片（一般節奏）
  • 5 分鐘報告 → 建議 8-10 張 ／ 10 分鐘報告 → 建議 12-15 張
簡報頁面類型建議視覺風格效果
封面與大標題頁日系清新風格 或 3D 黏土質感建立第一眼好感度
數據與績效頁現代數據儀表板風強調 KPI、折線圖、圓餅圖
作業流程/步驟頁等距視角微縮 或 蛇形流程讓複雜作業程序立體易懂
多項重點並陳頁便當盒排版法模組化切分，資訊豐富不雜亂
※ 搭配 資訊圖 Prompt 生成器 ↗ 使用效果更佳
📌 Prompt1 — 大綱校準（簡單版）
現在進入【格式骨架校準階段】。 請根據我們剛才完成的「【隼鷹計畫】新機型資安防火牆推廣與售後服務方案」報告， 針對【投資人與經銷商客戶】擬定簡報大綱， 語氣可以生動一點，畢竟「隼鷹」是這次的主角，別把牠寫得像一顆冰冷的鐵盒子。 規範： - 規劃約【10】張架構 - 每頁須含主題與核心觀點，禁止跳級產出內文或旁白 - 數據頁面請標註建議採用「便當盒排版法」 - 僅輸出邏輯骨架後停止，等我確認後再繼續 - 強制使用 Markdown code block 區塊輸出
📌 Prompt2 — 大綱校準（完整版，對應視覺規格）
# 任務進度：【簡報格式骨架校準階段】 ## 銜接背景與角色設定 - 資料來源：依據我們剛才完成的「【隼鷹計畫】新機型資安防火牆推廣與售後服務方案」報告 - AI 角色：資深業務幕僚，協助將報告轉化為具說服力的簡報邏輯 - 簡報對象：投資人與經銷商客戶，偏好數據導向、重視執行可行性 ## 簡報規格參數 - 上台時間：15 分鐘 - 投影片張數：約 10 張 - 輸出格式：以「投影片 [編號]：[標題]」呈現 - 每頁必含：核心論點摘要 + 建議視覺化圖型 ## 參、視覺與版面規格 - 整體風格：現代數據儀表板風（淺色介面、專業 KPI 數字卡片） - 色系：三色限定 主色：深海藍 #1E3A5F 輔色：香檳金 #C9A84C 底色：暖米白 #F5F0E8 - 視覺質感：無陰影、無漸層、幾何平整 - 封面設計：強制預留上方 30% 標題空白區 - 內頁排版：複雜數據頁標註「便當盒排版法」或「網格矩陣」 ## 強制約束 - 禁止產出具體簡報文案、旁白或生圖指令 - 僅輸出邏輯骨架，產出後立即停止，等待指令 ## 格式鎖定 - 強制以 Markdown 程式碼區塊輸出大綱 「報告背景已備齊，請開始擬定簡報邏輯大綱。」

3 NotebookLM 文件智慧整合應用
📌 延續【隼鷹計畫】——把新一代資安防火牆的規格書、業務會議記錄、客戶問題都丟進NotebookLM，用以下 Prompt 進行跨文件查詢與分析。NotebookLM 只根據你上傳的文件回答，不會自行腦補或引用外部資料——答案有所本，不是 AI 憑空生成的。
文件摘要與智慧查詢
使用時機：手邊有【隼鷹計畫】的產品規格書 PDF、幾份技術白皮書、幾個月的業務會議記錄，要從裡面找一個答案，光是翻資料就要花掉半小時——上傳至 NotebookLM 後直接用自然語言提問即可。
📌 Prompt — 直接提問範例
【隼鷹計畫】這款新一代資安防火牆的主要技術規格是什麼？ 哪幾份文件有提到【隼鷹計畫】的保固條款？ 這三個月的業務週報，經銷商對【隼鷹計畫】提出的問題，有哪些重複出現？
適用場景：產品規格查詢、歷次會議決議追蹤、業務報告重點整理、跨月份數據比較
會議錄音整理
兩步驟流程：Step 1 用 Gemini 將【隼鷹計畫】客戶說明會的錄音檔轉為逐字稿並整理成正式會議紀錄；Step 2 上傳至 NotebookLM 與其他文件放同一知識庫，未來可跨會議提問。
📌 Prompt — 逐字稿轉會議紀錄
這是一份【隼鷹計畫】客戶說明會的錄音逐字稿，語氣口語，內容較雜亂。 請協助完成以下整理： 1. 整理成正式會議紀錄格式，包含： 討論事項、各部門意見摘要、決議事項、後續追蹤事項 2. 找出所有主管交辦事項，列出負責人與預計完成時間 3. 整理「問題與改善建議」清單，標注是哪個部門提出的
關鍵觀念：錄音整理不是終點，而是起點。整理好的紀錄進入知識庫，才能被反覆查詢、持續產生價值。
適用場景：業務週會、客戶會議記錄整理、跨部門協調會議、主管指示追蹤
Podcast 語音摘要
使用時機：一份 20 頁的【隼鷹計畫】技術規格書、一本新的產品操作手冊，要靜下心來完整閱讀並不容易。NotebookLM 的 Audio Overview（語音總覽）功能，能將文件自動轉換為兩人對話式的 Podcast 摘要。
價值說明
理解節奏聽 AI 如何把複雜內容說得清楚易懂，學習表達邏輯
通勤學習把【隼鷹計畫】技術文件變成可以在移動中吸收的內容
簡報預演了解一份文件的重點應該用什麼順序說明
適用場景：新產品上市前的快速理解、新進人員產品知識熟悉、簡報內容預先消化、出差通勤時的知識補充
建立內部知識庫
使用時機：【隼鷹計畫】的SOP、過去類似資安產品的技術案例、常用的產品規格散落各處，新進同仁要花很長時間才能上手。NotebookLM 可以作為輕量型的部門知識庫。
  • 新進人員可以直接提問，不需要一直問資深同事
  • 業務／工程人員可以快速找到歷史案例與前例
  • 主管可以跨文件查詢，快速掌握業務全貌
關鍵觀念：NotebookLM 不是搜尋引擎，是會回答問題的知識助理，文件上傳得愈完整，回答得愈準確、愈有用。
適用場景：產品規格查詢與解釋、SOP 手冊整理、業務交接文件管理、跨部門案例比較與參考
4 AI 與 Excel 資料分析應用
📌 工作中每天面對大量表格、統計數字與業務報表。AI 的加入，不是要取代 Excel，而是讓你用說的，就能得到你要的結果——不需要記公式、不需要懂統計，只要說清楚你想做什麼，AI 幫你完成。
📌 Prompt — 不同主題同一對話建議
以下對話詢問以主題詢問，如果出現 【主題】字樣，表示為新主題與前後對話無關。
照片／紙本資料轉 Excel
操作習慣：整理完成後，利用【Ctrl + C】複製 AI 回覆的表格，再到 Excel 按【Ctrl + V】貼上

延伸練習：排列圖（Pareto）找出關鍵問題原因
📁 練習檔案：經銷商客戶滿意度調查.xlsx「排列圖」工作表（6筆，設備維修/退貨原因統計，含發生次數、累進比、佔比）
📌 排列圖（Pareto圖）的核心概念：找出佔問題80%的那20%關鍵原因，優先處理，而不是每個問題都平均施力。
80/20法則是什麼？以本範例解讀
造成問題的原因裡，通常一小部分關鍵原因，就佔了大部分的影響
——不是每個原因都要平均分配資源去處理，要先抓「佔比最大的那幾個」優先解決。
原因發生次數累進比
韌體異常4233.6%
硬體故障3158.4%
安裝設定錯誤2477.6%
客戶操作不當1589.6%
相容性問題996.8%
包裝運送損壞4100%
6個原因裡，只要處理前3個（韌體異常、硬體故障、安裝設定錯誤），就已經涵蓋了77.6%的問題——
這3項只佔全部6項原因的一半（3/6），卻解決了將近八成的案例。
這就是80/20法則的意思：用20%的關鍵少數，解決80%的多數問題。
  • 資源有限時，優先處理「韌體異常」跟「硬體故障」，光這兩項就佔了58.4%，投資報酬率最高
  • 後面三項加起來才22.4%，即使花同樣力氣處理，效益也遠不如前面兩三項
  • 排列圖（Pareto圖）就是把發生次數由高到低排序、疊加累進比這條線，用視覺化方式一眼看出「關鍵少數」跟「次要多數」的分界點在哪裡

📌 提示詞方向（以季為例）
請根據這份2024-2025年的產品類別銷售資料（欄位含季別），設計一份互動式數據儀表板： - 加一個「季度拖拉滑桿」（2024Q1-2025Q4），拖到哪一季，所有卡片與圖表都要跟著變動 - 4張KPI卡片： ① 當季銷售金額最高的產品類別 ② 自2024Q1以來成長幅度最大的產品類別 ③ 自2024Q1以來降幅最大的產品類別 ④ 當季排行榜（5個產品類別依銷售金額由高到低排名） - 圖表：各產品類別銷售金額趨勢圖、當季排行橫條圖 - 下方自動產生資料表，含銷售數量、平均單價欄位
常見疑難排解：第一版生成後，若滑桿只控制得到資料表、卡片跟圖表沒有跟著動 → 補一句：「年月滑桿要控制所有卡片與圖表的數據，不是只有資料表；拖拉時全部要跟著變化；加一個播放鈕可以自動跳月份，暫停鈕可以隨時停」→ 重新生成即可修正
「自動更新」的原理：資料來源本身連的是Google試算表（不是靜態貼上去的數字）。直接到Google試算表新增/修改一列資料，回到儀表板按「重新整理」，畫面就會反映最新的資料——也可以設定排程（每隔幾分鐘/幾秒自動更新一次）。
巨集1：合併多個檔案
工作情境：每月各站別分別繳交獨立的Excel報表，需手動開啟、複製、貼上，耗時容易出錯。透過AI生成VBA巨集，自動將同一資料夾內所有Excel檔案合併到一張工作表。
📁 練習檔案：新竹站業績.xlsx（14筆）／台北站業績.xlsx（13筆）／台中站業績.xlsx（13筆）— 三份欄位結構一致，合計40筆
📌 AI 生成VBA巨集的Prompt範例
請幫我寫一段Excel VBA巨集， 功能：將同一個資料夾內所有.xlsx檔案的第一張工作表資料， 合併到目前開啟的Excel檔案中的「彙整」工作表， 每個檔案的第一列（標題列）只保留一次，不重複。
📌 AI 回傳的巨集範例（VBA）
Sub MergeAllFiles() Dim folderPath As String Dim fileName As String Dim wbSource As Workbook Dim wsTarget As Worksheet Dim wsSource As Worksheet Dim lastRow As Long Dim isFirst As Boolean folderPath = "C:\彙整資料夾\" ' ← 修改為你的資料夾路徑 isFirst = True Set wsTarget = ThisWorkbook.Sheets("彙整") wsTarget.Cells.Clear fileName = Dir(folderPath & "*.xlsx") Do While fileName <> "" Set wbSource = Workbooks.Open(folderPath & fileName) Set wsSource = wbSource.Sheets(1) lastRow = wsSource.Cells(wsSource.Rows.Count, 1).End(xlUp).Row If isFirst Then wsSource.Rows(1).Copy wsTarget.Rows(1) ' 複製標題 isFirst = False End If Dim targetLastRow As Long targetLastRow = wsTarget.Cells(wsTarget.Rows.Count, 1).End(xlUp).Row + 1 wsSource.Rows("2:" & lastRow).Copy wsTarget.Rows(targetLastRow) wbSource.Close False fileName = Dir Loop MsgBox "合併完成！" End Sub
📌 如何在Excel套用巨集
S1. 複製AI給的VBA程式碼 S2. 先在Excel建立一張名為「彙整」的工作表 S3. 修改程式碼第9行的資料夾路徑為你的實際路徑 S4. Excel【開發人員】索引標籤 → 點【Visual Basic】 S5. 插入 → 模組 → 貼上程式碼 S6. 按【F5】或點【執行】→ 完成
狀況原因解法
找不到「彙整」工作表工作表名稱不符確認工作表名稱完全一致（含全形/半形）
路徑錯誤無法執行路徑格式不對結尾必須有【\】，路徑不可含特殊字元
標題重複出現每個檔案的首列都被複製巨集已處理，確認AI給的版本含【isFirst】判斷
只合併了部分檔案資料夾內有非.xlsx格式請AI加入.xls或.csv的處理版本
巨集2：依欄位拆分成多個檔案
工作情境：主管手上是一份全區彙整的訂單總表，但要分別寄給新竹、台北、台中、高雄四個站的主管看各自站別的資料，需要手動篩選、複製、貼上、分別存檔，四個站就要重複四次。
📁 練習檔案：全區業績彙整_含分區.xlsx（52筆，欄位：訂單編號、業務代表、產品名稱、數量、金額、下單日期、分區）—— 這正好是前一個「合併」巨集的反向操作
📌 AI 詢問
請幫我寫一段Excel VBA巨集， 功能：將目前工作表中的資料，依【分區】欄位的內容分類， 把每個分區的資料各自存成一個獨立的Excel檔案， 檔名依分區命名（例如：新竹.xlsx、台北.xlsx）， 每個檔案都要保留標題列，並存放在同一個資料夾內。
📌 AI 回傳的巨集範例（VBA）
Sub SplitByRegion() Dim ws As Worksheet Dim lastRow As Long, lastCol As Long Dim i As Long, c As Integer Dim regionCol As Integer Dim regionName As String Dim outputPath As String Dim dict As Object Set dict = CreateObject("Scripting.Dictionary") Set ws = ThisWorkbook.Sheets("彙整") ' ← 修改為你的工作表名稱 lastRow = ws.Cells(ws.Rows.Count, 1).End(xlUp).Row lastCol = ws.Cells(1, ws.Columns.Count).End(xlToLeft).Column regionCol = 0 For c = 1 To lastCol If ws.Cells(1, c).Value = "分區" Then regionCol = c Exit For End If Next c If regionCol = 0 Then MsgBox "找不到「分區」欄位" Exit Sub End If outputPath = ThisWorkbook.Path & "\拆分結果\" If Dir(outputPath, vbDirectory) = "" Then MkDir outputPath Dim wb As Workbook Dim wsNew As Worksheet Dim targetRow As Long For i = 2 To lastRow regionName = ws.Cells(i, regionCol).Value If Not dict.exists(regionName) Then Set wb = Workbooks.Add Set wsNew = wb.Sheets(1) ws.Rows(1).Copy wsNew.Rows(1) dict.Add regionName, wb End If Set wb = dict(regionName) Set wsNew = wb.Sheets(1) targetRow = wsNew.Cells(wsNew.Rows.Count, 1).End(xlUp).Row + 1 ws.Rows(i).Copy wsNew.Rows(targetRow) Next i Dim key As Variant For Each key In dict.Keys dict(key).SaveAs outputPath & key & ".xlsx", FileFormat:=51 dict(key).Close False Next key MsgBox "拆分完成！檔案已存放於：" & outputPath End Sub
狀況原因解法
找不到「分區」欄位欄位名稱打錯或位置不同確認標題列文字完全一致，或請AI改成依欄位位置判斷
檔名出現亂碼或無法存檔分區欄位內容含特殊符號/空白請AI加入檔名清理邏輯，過濾掉不可用於檔名的符號
部分分區資料被合併到同一檔案分區欄位內容格式不一致（多空格）先用TRIM()清理分區欄位，或請AI在巨集中加入Trim處理
執行後沒有跳出「拆分結果」資料夾巨集尚未存檔的活頁簿沒有Path先將主檔案存檔一次，再執行巨集
