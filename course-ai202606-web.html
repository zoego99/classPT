課程單元（模組一～七）
1一頁式網頁該有哪些區塊
📌 先搞懂類型再動手做，後面的Prompt才知道為什麼要那些區塊
核心觀念：
一頁式網頁不是「隨便排幾個區塊」，而是先確定這個頁面的目的——是純粹介紹一件事，還是要讓使用者採取某個行動。
目的不同，需要的區塊跟結尾方式也不一樣。
商品詳情頁（ 例如：寵物商品，純展示型 ）
📌 目的：
  • 把一項商品的賣點、外觀、規格講清楚，讓使用者「認識並信任」這個商品，不強制要求下一步動作。
  • 這是純展示型的用法（ 本課程的寵物商品範例即屬此類 ）。
常見區塊作用
Hero（ Slider輪播 ）第一眼視覺，快速傳達商品調性
痛點共鳴描述使用者常見困擾，引起共感
產品特色條列賣點，說明「為什麼這個商品值得注意」
使用情境生活化描述，幫助想像實際使用畫面
規格 / 成分表提供具體、可查證的資訊
顧客見證用他人經驗建立信任感
FAQ ( 常見問答 )預先解答疑慮（詳見下方名詞說明）
活動行銷頁（ 例如：咖啡館開幕 ）
📌 目的：
  • 導引使用者採取一個「有時間限制」的具體行動——通常是報名或參加。
  • 急迫感（限量、早鳥）是這類頁面的重點，結尾一定要有明確的行動按鈕。
常見區塊作用
Hero（ Slider輪播 ）疊加活動主題、時間地點
活動亮點/流程說明當天會發生什麼事
早鳥優惠/限量名額製造急迫感，加速決定
報名區塊（ TA 行動呼籲 ）「立即報名」按鈕＋LINE QRcode，是整頁的行動終點
FAQ ( 常見問答 )預先解答疑慮，減少猶豫
交通/地圖資訊降低到場的實際障礙
形象展示＋導流型 ( 餐廳行銷 )
📌 餐廳其實是前兩種的「混合型」
  • 一半像商品詳情頁——需要把餐廳形象、菜色、氛圍講清楚，建立信任感
  • 一半像活動行銷頁——結尾一樣需要明確的行動終點
  • 只是這裡的行動是「訂位」，而不是「報名」，而且沒有時間限制（常態性經營，不是限時活動）
名詞說明：FAQ 與 CTA
FAQ（Frequently Asked Questions，常見問答）：
把使用者常見的疑慮預先整理成問答形式，放在頁面後段，
目的：減少使用者需要另外詢問或猶豫不決的機率（例如：營業時間、是否可攜寵、如何退換貨）。

CTA（Call To Action，行動呼籲）：
泛指頁面上「引導使用者做出下一步動作」的按鈕或文字，例如「立即報名」「立即訂位」「加入LINE好友」。
不是每個頁面都需要CTA——
寵物商品頁是刻意不放CTA的純展示型，因為目的只是形象曝光，不需要導引任何後續動作；
活動行銷頁與餐廳頁則一定需要CTA，因為它們的目標就是要使用者採取行動（報名／訂位）。

5GitHub Pages 部署
📌 免費、需要GitHub帳號，適合想順便學版本控管概念的學生
建立Repo與上傳
步驟操作重點
1. 註冊/登入GitHub前往 github.com 註冊免費帳號
2. 建立新Repository點右上角「+」→「New repository」，命名（例如：my-pet-page），設定為Public，勾選「Add a README file」
3. 上傳檔案進入該Repo頁面，點「Add file」→「Upload files」，把Gemini生成的.html檔案拖曳上傳
4. 檔名務必改為 index.htmlGitHub Pages預設只會找 index.html 當首頁，檔名不對網頁打不開
5. Commit確認捲到底部點「Commit changes」完成上傳
啟用Pages
步驟操作重點
1. 進入SettingsRepo頁面上方點「Settings」
2. 找到Pages選項左側選單點「Pages」
3. 設定BranchSource選擇「Deploy from a branch」，Branch選「main」，資料夾選「/ (root)」，點Save
4. 取得網址等待約1~2分鐘，頁面會顯示網址，格式為 https://你的帳號.github.io/repo名稱/
常見問題：網頁打開是空白或404，通常是檔名沒改成index.html，或是Branch/資料夾設定錯誤，回頭檢查這兩點即可。
6Netlify 部署
📌 免費、不需要學版本控管，最快上線方式，適合時間有限的教學場景
拖曳部署
步驟操作重點
1. 註冊/登入Netlify前往 netlify.com 註冊免費帳號（可用Email或Google帳號登入）
2. 進入部署頁面登入後畫面通常會看到「Drag and drop your site output folder here」的拖曳區塊
3. 拖曳資料夾把包含.html檔案的資料夾直接拖進網頁畫面（不需要壓縮成zip，也不需要改檔名為index.html——Netlify會自動辨識）
4. 取得網址幾秒鐘內自動部署完成，畫面會顯示一組隨機網址（例如 random-name-123.netlify.app）
與GitHub Pages的差異：
Netlify不需要建Repo、不需要懂版本控管概念，拖曳資料夾即可上線，是三種部署方式裡最快的一種
缺點是隨機網址不好記，需要的話可以在後台改成自訂的子網域名稱。
更新網頁內容

➜ 之後如果內容有修改（例如：把佔位的Google表單連結、LINE QRcode圖片換成真實資料），只要把修改後的資料夾重新拖曳一次到同一個網站的部署頁面，Netlify會自動覆蓋更新，網址不會改變。

7HTML基本結構
📌 目的：讓學生看得懂Gemini生成的程式碼在寫什麼，才知道怎麼自己微調
網頁基本骨架
一個HTML檔案由標籤（Tag）構成，語法是 <標籤名稱>內容</標籤名稱>。標籤可以加屬性，例如 <a href="網址">文字</a>，href就是屬性。
區塊說明
<!DOCTYPE html>宣告這是HTML5文件，一定要放在最上面
<html lang="zh-Hant-TW">整份文件的根標籤，lang標示語系（繁體中文）
<head>...</head>放「看不見」的設定：網頁標題(title)、字元編碼(charset)、手機顯示比例(viewport)
<body>...</body>放「看得見」的所有內容：文字、圖片、按鈕、區塊，都在這裡面
<style>...</style>網頁元素美化
<script>...</script>網頁互動邏輯
常用標籤
標籤用途
<h1>～<h6>標題，數字越小字越大越重要，h1建議整頁只用一次
<p>段落文字
<img src="圖片路徑" alt="說明文字">放圖片，src是圖片位置，alt是圖片說明
